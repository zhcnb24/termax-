# v0.1.1 版本说明

## 版本定位

`v0.1.1` 是一次以发布落地和文档规范补齐为主的小版本更新。

这次更新的重点不是新增大量业务能力，而是把“能构建、能发布、能追踪”三件事真正落地。

## 本次更新

### 已完成

- 生成新的 `release` 签名证书并完成 APK 签名
- 构建成功 `Release APK`
- 推送版本更新到仓库 `main`
- 创建 GitHub Release 并上传 APK 与校验文件
- 新增项目规则库与 AI 检索入口
- 为后续每次更新建立固定文档归档方式

### 版本产物

- Release：`v0.1.1`
- APK：`termax-v0.1.1-release.apk`
- SHA256：`9d6941165669fc9272c29c6cff2fcd8334f5813178640ab779a8eff6067832b8`

## 为什么更新

此前项目虽然已有 Android 源码和基本文档，但还缺少以下关键能力：

- 正式签名发布链路
- 版本可追踪文档体系
- AI 可检索的规则入口
- 更新说明的固定落点

`v0.1.1` 的意义就是把这些基础设施补齐。

## 当前状态

当前项目仍处于第二阶段 MVP 范围内，已经具备：

- 基础 AI 聊天
- 基础任务记录
- 基础环境检测
- 基础 Shell 执行
- 基础文件工作区

但尚未进入完整 Agent 自动执行阶段。

## 下一步建议

建议优先推进：

1. `Agent Planner`
2. `Guard`
3. `Verifier`
4. `Repairer`
5. 真正的 Termux / Shizuku 执行接入
