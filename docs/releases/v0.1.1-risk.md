# v0.1.1 风险与影响说明

## 1. 可能会发生什么

### 签名相关

- 本次 `Release APK` 使用了新生成的签名证书
- 如果后续丢失该 `keystore`，将无法继续以同一签名更新正式安装包

### 构建相关

- 当前构建依赖代理和镜像仓库可用性
- 如果环境无法访问默认仓库，构建仍可能受网络条件影响

### 功能相关

- 当前版本仍不是完整 Agent 执行版
- 用户可能误以为已经具备自动拆任务、自动修复、自动持续执行的全部能力

### 平台相关

- 虽然目标兼容 Android 8.0 及以上，但当前仍需真机进一步验证
- 不同设备厂商对后台任务、通知、权限的限制可能影响后续体验

## 2. 当前主要缺口

- 真正的 Agent 执行闭环尚未完成
- Termux / Shizuku 仍以检测和设计为主
- 沙盒、快照、回滚仍是规则与架构层设计，未完整实现

## 3. 建议规避方式

- 妥善保存签名文件
- 后续每次发布继续保留版本记录
- 对外说明时明确区分“已实现能力”和“规划能力”
- 在进入第三阶段前优先补齐执行闭环与风险确认机制

## 4. 回滚说明

代码层：

- 可通过 Git 回退到 `v0.1.0` 或更早提交

发布层：

- 可下架 `v0.1.1` Release 资产
- 但如果用户已安装当前签名的正式包，后续回滚仍需使用同一签名体系
