# ADR 目录说明

ADR 是 Architecture Decision Record，即架构决策记录。

本目录用于记录以下类型的重要决策：

- 为什么选择某种模块划分
- 为什么采用某种模型路由方案
- 为什么采用某种安全或权限策略
- 为什么在某次版本中调整构建、发布、签名或仓库结构

## 建议命名

建议未来使用以下格式命名：

- `0001-xxx.md`
- `0002-xxx.md`

例如：

- `0001-release-signing-strategy.md`
- `0002-docs-traceability-strategy.md`

## 建议结构

每份 ADR 建议包含：

- 背景
- 决策内容
- 备选方案
- 影响
- 风险
- 结论
