# 产品总纲

## 1. 项目名称

- 中文名：`Termux千秋辅助`
- 英文名：`Termux QianQiu Assistant`

## 2. 项目定位

本项目是一个面向 Android 平台的超级 AI Agent 应用。

产品定位不是普通聊天 AI，也不是普通 Termux 工具箱，而是一个以 Termux 为执行核心的 Android 智能开发平台。

目标融合以下能力特征：

- Termux
- AI Agent
- Cursor
- Claude Code
- Manus

核心理念：

> 纵使前方万般难，踏步前行入平川。

## 3. 目标平台

- Android 8.0 及以上
- 兼容较新的 Android 系统版本
- 优先支持 `ARM64`
- 支持 `ARMv7`
- `x86_64` 为可选兼容目标

## 4. 产品原则

- 以聊天为主入口
- 以 Agent 为任务调度核心
- 以 Termux 为主要执行环境
- 以安全、可追溯、可回滚为底层约束
- 以长期演进和模块化为工程方向
- 永久免费，不得引入会员系统或付费墙

## 5. UI 与交互方向

整体风格：

- 70% Telegram
- 20% Cursor
- 10% VSCode

视觉和交互要求：

- 深色模式优先
- Telegram 新版信息流风格
- iOS 风格动画与交互
- 二次元元素融合
- 高级磨砂玻璃效果
- 大圆角
- 流畅动效

## 6. AI 角色系统

默认角色名称：

- `千秋`

默认设定：

- 日常模式
  - 温柔
  - 陪伴型
  - 二次元少女
- 工作模式
  - 专业工程师
  - 系统架构师
  - AI Agent 执行官

长期支持：

- 自定义人格
- 自定义立绘
- 自定义声音
- 角色卡导入
- JSON 角色
- PNG 角色卡

## 7. 登录与同步

支持：

- 手机号登录
- 邮箱登录
- 第三方登录

预留：

- 云同步接口
- 用户数据同步接口

## 8. AI 核心能力目标

用户输入目标后，AI 需要逐步具备以下能力：

1. 分析目标
2. 判断可行性
3. 联网搜索方案
4. 自动拆分任务
5. 自动生成代码
6. 自动执行代码
7. 自动读取结果
8. 自动修复错误
9. 自动继续执行
10. 直到任务完成

## 9. 工作模式

- 普通模式
  - 危险操作前确认
- 专家模式
  - 自动执行常规高频流程
- 开发者模式
  - 开放高级控制和更多日志

## 10. 关键系统范围

产品长期应覆盖以下系统：

- Termux 集成
- 文件系统能力
- 安全规则
- 长期记忆系统
- 知识库系统
- AI 模型系统
- 代码工作台
- 任务中心
- 通知系统
- 插件市场
- 沙盒与快照系统
- AI 行动日志
- 语音系统
- 多语言系统

## 11. 页面结构

- 首页
- 个人中心
- AI聊天
- 代码工作台
- 知识库
- 任务中心
- 插件市场
- 设置中心

## 12. 开发要求

需要长期输出并持续维护以下成果：

- Android 客户端源码
- 后端架构设计
- 数据库设计
- AI 模块设计
- Agent 模块设计
- 插件系统设计
- 页面原型设计
- 项目目录结构设计

优先目标：

- 先保证架构完整性
- 再保证可扩展性
- 最后持续优化体验和高级能力

## 13. 文档与追踪原则

本项目要求“一切有迹可循”。

因此后续每次更新都必须保留：

- 本次更新说明
- 改动文件清单
- 尚未完成项
- 后续建议
- 已知风险
- 可能影响
- 回滚说明或回滚条件
