# 安全规则

## 1. 默认目标

本项目追求强执行能力，但执行能力必须建立在安全边界之上。

## 2. 默认禁止事项

默认禁止以下操作：

- 删除系统文件
- 修改系统关键配置
- 删除用户重要数据
- 未经确认执行高破坏性命令
- 未经确认授予高权限插件能力

## 3. 高风险操作

以下操作应视为高风险：

- 批量删除
- 覆盖关键配置
- 更改签名、包名、数据库结构
- 请求 Shizuku 高权限
- 请求 Root 相关能力
- 安装高权限插件

## 4. 高风险处理原则

高风险操作前尽量满足以下条件：

1. 给出风险说明
2. 告知影响范围
3. 说明是否可回滚
4. 提供确认步骤
5. 记录执行结果

## 5. Termux 与权限策略

执行优先级：

1. Shizuku
2. Termux API
3. Intent

但优先级不等于可以无条件执行。权限越高，确认和审计要求越高。

## 6. 插件权限等级

- 一级：读取文件
- 二级：修改文件
- 三级：执行命令
- 四级：Shizuku 权限
- 五级：Root 权限

插件安装流程至少包含：

1. 风险分析
2. 用户确认
3. 沙盒验证
4. 再进入正式环境

## 7. 沙盒与快照要求

重要任务尽量先进入沙盒执行区验证。

满足条件时应支持：

- 执行前备份
- 快照生成
- 一键恢复
- 任务回滚

## 8. 审计要求

涉及实际执行的任务，尽量记录：

- 做了什么
- 为什么这样做
- 影响了什么
- 输出结果是什么
- 是否触发了风险规则
