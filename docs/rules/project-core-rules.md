# 项目基础规则

## 1. 项目身份

`Termux千秋辅助` 是一个面向 Android 的超级 AI Agent 平台，不是单纯聊天应用，也不是单纯 Termux 工具集合。

## 2. 核心方向

必须始终坚持以下主线：

- 聊天是入口
- Agent 是中枢
- Termux 是执行核心
- 安全、追踪、回滚是底层保障

## 3. 产品红线

以下规则默认不可突破：

- 不得设计会员系统
- 不得设计付费墙
- 不得把核心功能建立在强制付费之上
- 不得偏离 Android 端优先的产品目标
- 不得把项目退化为普通聊天壳或普通命令工具壳

## 4. 研发优先级

新增需求出现时，优先级判断顺序如下：

1. 是否强化 Agent 能力
2. 是否强化执行能力
3. 是否强化安全与可追溯能力
4. 是否强化知识库与长期记忆能力
5. 是否强化 UI 体验和易用性

## 5. 文档优先原则

以下内容必须优先沉淀到文档，而不是只留在聊天记录里：

- 产品定位
- 行为规则
- 安全边界
- 更新规范
- 架构决策
- 版本变更记录

## 6. 开发原则

- 先保证骨架正确，再补能力深度
- 先保证可编译，再追求大而全
- 先保证模块边界清晰，再扩展新功能
- 先保证追踪清晰，再推进复杂自动化

## 7. 交付原则

每次交付都应尽量回答清楚：

- 做了什么
- 没做什么
- 为什么这样做
- 风险在哪里
- 下一步建议是什么
