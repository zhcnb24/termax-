# 文档总览

本目录用于沉淀 `Termux千秋辅助` 的长期产品上下文、规则约束、版本追踪和架构说明。

目标不是堆积文档，而是让后续每一次开发都能做到：

- 有统一产品目标
- 有统一执行规则
- 有统一安全边界
- 有统一版本追踪
- 有统一 AI 检索入口

## 目录说明

### `product/`

产品总纲、定位、目标用户、页面结构、能力边界、长期愿景。

### `rules/`

项目基础规则、AI 执行规则、安全规则、更新规则。

### `context/`

给 AI 检索使用的入口文档和上下文索引，帮助快速定位规则和设计依据。

### `releases/`

每次版本更新的说明、清单、风险、改动文件索引。

### `adr/`

架构决策记录，用于回答“为什么这样做，而不是那样做”。

### 根目录已有文档

- `architecture.md`
  - 当前总体架构设计
- `phase2-roadmap.md`
  - 第二阶段开发范围与已实现情况

## 文档使用原则

1. 新功能开始前，先查 `context/ai-retrieval-index.md`
2. 涉及行为约束时，优先查 `rules/`
3. 涉及产品定位时，优先查 `product/`
4. 涉及历史版本差异时，优先查 `releases/`
5. 涉及重要方案选择时，新增或补充 `adr/`

## 更新要求

每次正式更新至少补充以下信息：

- 更新了什么
- 为什么更新
- 影响了哪些文件
- 还差什么
- 建议什么
- 可能会发生什么
- 是否可回滚

详细规则见 `rules/update-rules.md`。
